Represents a shared variant value.A Value object contains a reference to a var object, and can get and set its value. Listeners can be attached to be told when the value is changed.The Value class is a wrapper around a shared, referencecounted underlying data object this means that multiple Value objects can all refer to the same piece of data, allowing all of them to be notified when any of them changes it.When you create a Value with its default constructor, it acts as a wrapper around a simple var object, but by creating a Value that refers to a custom subclass of ValueSource, you can map the Value onto any kind of underlying data.Important note! The Value class is not threadsafe! If you're accessing one from multiple threads, then you'll need to use your own synchronisation around any code that accesses it.
